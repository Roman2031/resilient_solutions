**Dimensions**: 1024 x 500 pixels

**File size**: Up to 1 MB

## Assets Checklist

### Required Assets

- [ ] App Icon (1024x1024, no transparency, no rounded corners)
- [ ] Feature Graphic (1024x500, Play Store only)
- [ ] Phone Screenshots (min 2, max 8)
- [ ] Tablet Screenshots (if supporting tablets)
- [ ] App Preview Video (optional but recommended)

### Screenshot Content Guidelines

1. **Login Screen**
   - Show secure login options
   - Highlight authentication methods
   - Professional and clean

2. **Dashboard**
   - Display key metrics
   - Show navigation
   - Demonstrate main features

3. **Circles**
   - Show circle list
   - Display circle details
   - Highlight member management

4. **Courses**
   - Course catalog grid
   - Course details
   - Enrollment process

5. **Video Player**
   - Playing a lesson
   - Show controls
   - Display progress

6. **Messaging**
   - Conversation list
   - Chat interface
   - Group messaging

7. **Notes**
   - Note-taking interface
   - Note list
   - Action items

8. **Admin Portal**
   - Analytics dashboard
   - User management
   - System overview

### Screenshot Best Practices

- Use high-quality device frames
- Show actual app content (no mockups)
- Ensure text is readable
- Consistent UI across screenshots
- Show diverse user scenarios
- Highlight key features
- Professional appearance
- No sensitive/personal data
- Updated with latest UI

## Privacy Policy

Must include information about:

1. **Data Collection**
   - User authentication data (email, name)
   - Usage analytics
   - Device information

2. **Data Usage**
   - App functionality
   - Service improvement
   - Communication

3. **Data Sharing**
   - No third-party sharing
   - Except service providers (Firebase, etc.)

4. **User Rights**
   - Data access
   - Data deletion
   - Account closure

5. **Security**
   - Encryption
   - Secure storage
   - Access controls

6. **Contact Information**
   - Email: privacy@kingdominc.com
   - Website: www.kingdominc.com

## Testing Before Submission

- [ ] Install on test devices
- [ ] Complete user flow test
- [ ] Check all features work
- [ ] Verify no crashes
- [ ] Test on slow network
- [ ] Verify offline functionality
- [ ] Check all links work
- [ ] Verify proper error handling
- [ ] Test push notifications
- [ ] Verify in-app purchases (if any)

## Submission Process

### Google Play Console

1. Create app listing
2. Upload builds to Internal Testing
3. Complete store listing
4. Add pricing & distribution
5. Submit for review
6. Publish to production

### App Store Connect

1. Create new app
2. Upload build via Xcode
3. Complete app information
4. Add screenshots and media
5. Submit for review
6. Release when approved

## Review Guidelines Compliance

### Google Play Policies

- User data policy compliant
- Family-friendly content
- No misleading information
- Functional and stable
- Appropriate content rating

### App Store Review Guidelines

- Complete and functional
- Accurate metadata
- Business model clear
- Privacy policy present
- Content appropriate
- Performance standards met
